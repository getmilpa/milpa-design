# Milpa Design 0.8.0 «el rocío» — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the three brand-theming gaps the Milpa⇄Brutalist proof exposed — `--border-style` (wasn't a token), themeable backdrop (`--surface-backdrop` + `--blur-*`), and `verify-theme` translucency (parse alpha + composite over `--bg`) — then re-run the proof with glassmorphism as a 3rd flavor.

**Architecture:** Additive tokens authored in `tokens/milpa-tokens.json`; `build-tokens.mjs` generates `dist/milpa-tokens.css` + `dist/tailwind.config.js` + `theme.contract.json` (drift-gated). New tokens default to the current literal (`solid`/`none`/current blur px) → default render pixel-identical. `verify-theme.mjs` gains alpha color parsing + alpha-compositing translucent backgrounds over `--bg` before the WCAG check. The glass flavor is a token-only skin exercising the new surface.

**Tech Stack:** DTCG JSON + zero-dep Node generator/verifier; pure CSS (`@layer`, `backdrop-filter`); `npm test`; browser computed-style diffing.

## Global Constraints

- **Zero visual change by default:** `--border-style: solid`, `--surface-backdrop: none`, `--blur-sm/base/lg` = the current hardcoded `2px`/`4px`/`10px`. Default build pixel-identical — verified by `getComputedStyle` diff = 0 across the 6 shipped proofs (dark+light).

- New invariant: **`--bg` must be opaque** (it is the compositing reference for translucent surfaces). verify-theme errors if a skin sets a translucent `--bg`.
- All CSS in its `@layer`. Closed palette — this release adds NO colors → **0 new contrast pairs**. Zero published JS. Additive → **minor 0.8.0**. Contract-piece count stays **68**.

## Task 4: `verify-theme` translucency — alpha parse + composite over `--bg` (G3)

**Files:** Modify `scripts/verify-theme.mjs`; extend `scripts/verify-theme.test.mjs`; add `proof/skins/glass-broken-skin.css` fixture; Modify `THEMING.md`

**Interfaces:** Consumes `theme.contract.json` (invariant + contrast pairs, unchanged shape).

- [ ] **Step 1: Alpha-aware color parsing**

In `scripts/verify-theme.mjs`, replace the hex-only representation with `{r,g,b,a}` (0–255, a 0–1). Add:
```js
// --- parseo de color: #RRGGBB, #RRGGBBAA, rgb()/rgba() → {r,g,b,a} ---
function parseColor(s) {
  s = s.trim();
  let m = /^#([0-9a-fA-F]{2})([0-9a-fA-F]{2})([0-9a-fA-F]{2})([0-9a-fA-F]{2})?$/.exec(s);
  if (m) return { r: parseInt(m[1],16), g: parseInt(m[2],16), b: parseInt(m[3],16), a: m[4] !== undefined ? parseInt(m[4],16)/255 : 1 };
  m = /^rgba?\(\s*([\d.]+)[ ,]+([\d.]+)[ ,]+([\d.]+)(?:[ ,/]+([\d.]+%?))?\s*\)$/.exec(s);
  if (m) { const a = m[4] === undefined ? 1 : (m[4].endsWith('%') ? parseFloat(m[4])/100 : parseFloat(m[4])); return { r: +m[1], g: +m[2], b: +m[3], a }; }
  return null;
}
```
Widen the skin declaration capture (the second color regex) so it grabs `#RRGGBB`, `#RRGGBBAA`, and `rgb()/rgba()` values into the color map (store the raw string; parse with `parseColor` at use time). The DTCG defaults (`prim(ref)` → 6-hex) parse as `a:1`.

- [ ] **Step 2: Composite over `--bg` + adapt CR**

Add:
```js
const compose = (c, base) => c.a >= 1 ? c : {
  r: Math.round(c.r * c.a + base.r * (1 - c.a)),
  g: Math.round(c.g * c.a + base.g * (1 - c.a)),
  b: Math.round(c.b * c.a + base.b * (1 - c.a)), a: 1,
};
```
Change `Y`/`CR` to take `{r,g,b}` (0–255) instead of a hex string (normalize channels /255 inside). In the pair loop, per theme: resolve `--bg` (parseColor); **if `--bg.a < 1` → error + fail** (the opaque invariant). Resolve fg, bg (parseColor); if `bg.a < 1` compose it over the resolved `--bg`; if `fg.a < 1` compose it over that composited bg; then `CR(fgRGB, bgRGB)`. The pass/fail threshold logic is unchanged.

- [ ] **Step 3: Fixtures + regression test**

Create `proof/skins/glass-broken-skin.css`: a glass skin whose translucent surface composites to INSUFFICIENT contrast (e.g. `--surface: #FFFFFF14` over a light `--bg` with dark `--text` — the composite is near-white so light text fails; pick values that fail a real pair). Extend `scripts/verify-theme.test.mjs` with a case asserting `glass-broken-skin.css` → exit 1 (composited contrast fails), and (added in Task 5) the real `glass-skin.css` → exit 0.
