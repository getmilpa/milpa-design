# Milpa Design 0.7.0 «la piel» — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make theming release-grade: split the one UI font into `--font-heading`/`--font-body` (+`--font-serif`,`--weight-semibold`), tokenize the structural dimensions a skin can't currently reach (focus-ring, border-width, container, app-shell, measure), and make `verify-theme` validate the 7 token groups by form instead of gating color only — all additive, defaults = current values, zero visual change.

**Architecture:** Tokens are authored in `tokens/milpa-tokens.json` (DTCG) and `scripts/build-tokens.mjs` generates THREE drift-gated outputs: `dist/milpa-tokens.css`, `dist/tailwind.config.js`, `theme.contract.json`. New tokens go in the JSON + the generator; the published CSS then re-points its hardcoded literals to the new `var()`s. Because every new token defaults to the exact current literal, the default render is pixel-identical; the win is new override surface. `verify-theme.mjs` gains a form-validation pass over non-color tokens.

**Tech Stack:** DTCG JSON tokens + a zero-dependency Node generator/verifier; pure CSS (`@layer`); `npm test` (contrast + governance + `build-tokens --check` drift + layer-guard); browser computed-style diffing for the zero-change proof.

- [ ] **Step 2: Build a group→validator map + run form checks**

Add a validator map and check every set token that belongs to a known non-color group:
```js
// --- form validators por grupo (Task 1 del contrato: validation.form.types) ---
const LEN = /^-?(\d*\.?\d+)(px|rem|em|ch|vw|vh|%)$|^0$/;
const TIME = /^(\d*\.?\d+)(ms|s)$/;
const NUM = /^-?\d*\.?\d+$/;
const isLen = (v) => v.split(/\s+/).every((p) => LEN.test(p) || /^(min|max|calc|clamp)\(/.test(v));
const groupOf = (tokName) => {
  for (const [g, list] of Object.entries(contract.requiredTokens)) {
    if (g === 'color') continue;
    if (list.includes('--' + tokName)) return g;
  }
  return null;
};
const wellFormed = (g, name, v) => {
  if (v === '' ) return false;
  if (g === 'type') {
    if (name.startsWith('font-')) return v.length > 0;               // familia no-vacía
    if (name.startsWith('leading-')) return NUM.test(v);
    if (name.startsWith('weight-')) return NUM.test(v) && +v >= 1 && +v <= 1000;
    return isLen(v);                                                  // text-*, tracking-*
  }
  if (g === 'space' || g === 'radius' || g === 'size') return isLen(v) || v === '9999px';
  if (g === 'zIndex') return /^-?\d+$/.test(v);
  if (g === 'motion') {
    if (name.startsWith('dur-') || name.startsWith('stagger-')) return TIME.test(v);
    if (name.startsWith('ease-')) return /^(linear|cubic-bezier\(|steps\()/.test(v);
    return isLen(v);                                                  // rise-*
  }
  if (g === 'elevation') return v.length > 0;
  return true;
};
let formFails = 0;
for (const [name, v] of Object.entries(setTokens)) {
  const g = groupOf(name);
  if (!g) continue;                                                   // token desconocido o color → lo maneja el gate de contraste
  if (!wellFormed(g, name, v.trim())) { formFails++; console.log(`  FORM FAIL  --${name}: ${v.trim()}  (grupo ${g})`); }
}
```
Fold `formFails` into the exit condition: change the final `process.exit(fails === 0 ? 0 : 1)` to `process.exit(fails === 0 && formFails === 0 ? 0 : 1)`, and include `formFails` in the summary line (e.g. `... ${formFails} malformados`).
